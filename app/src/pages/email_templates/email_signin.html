Email Templates

This is the email sent to a user that will contain a link to sign in to the app. This requires coordination with the app itself to capture the link and process the token accordingly.

Note that the default email message contains a link with several token values; this link must be maintained in the email in order for the user to sign in.

However, you will need to replace the ${host} value with a custom host. If you used the default ${host} value it will conflict with other apps built on the Bridge server. The initial ${host} value is for testing purposes.

There are other values available:

${studyName}
the name you entered for your study
${sponsorName}
the name you entered for the group sponsoring the study
${supportEmail}
your support email address
