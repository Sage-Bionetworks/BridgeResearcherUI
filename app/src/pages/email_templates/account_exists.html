Email Templates

If a user tries to sign up twice for the same study, we will send this email. It notes the account already exists, and provides a link to reset the password (it seems likely the user has forgotten it).

It must contain the value ${resetPasswordUrl} or ${emailSignInUrl}. You cannot link these variables. Most web email clients will link it automatically.

There values that are available:

${resetPasswordUrl}
a link user can click to reset the password (older templates use ${url})
${emailSignInUrl}
a link user can click to sign in via email (only available if the study has email sign ins enabled)
${studyName}
the name you entered for your study
${sponsorName}
the name you entered for the group sponsoring the study
${supportEmail}
your support email address
