Verify email address or phone number

If the user signs up with an email address or phone number, we can send an email or SMS message to verify the user owns the email address or phone number. The account will not be enabled until this verification step is completed.

Account already exists

If a user sign ups for an app with a credential that is already registered in that the app (email or phone number), this message will be sent instead of a verification message.

Email Templates SMS Templates

Sign in

A sign in message can be sent via email or SMS. Either message will contain a link that can be intercepted by your mobile app. The app extracts a token in the link, and sends it back to the server to complete the sign-in operation.

Email Templates SMS Templates

Consent agreement

If we send a signed copy of a consent agreement to the user, this is the message that will be used to send it. For email the consent is attached, for SMS the consent will be provided for download via a link.

Email Templates SMS Templates

Reset password

When the user requests to change their password, this message is sent with a link the user can follow to reset their password. This will require opening a web browser.

Email Templates SMS Templates

Link to Install App

Some workflows supported by Bridge (such as the “Intent to Participate” APIs) will send a link to the user so they can download and install the app. This is the message used to send that link.
